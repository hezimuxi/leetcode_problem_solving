## 我到底菜在哪（展开讲讲）

为了避免误导路过的大佬，我决定继续坦白：

### 1) 数据结构：懂概念，不懂临场

- 链表：平时看着点头，写的时候手抖。
- 栈队列：知道原理，但题目一变形就懵。
- 哈希：会用，但经常漏掉键不存在的情况。

### 2) 算法思路：会一点，但不稳定

- 双指针：会写基础版，复杂一点就容易绕晕自己。
- 滑动窗口：窗口滑着滑着，边界就滑没了。
- 递归：逻辑想得挺美，终止条件经常写得不够严谨。

### 3) 代码细节：翻车概率偏高

- 边界条件漏判（空数组、单元素、重复值、越界）。
- 临时变量命名混乱，回头自己都不认识。
- “本地样例全过，提交瞬间报错”是常客。

### 4) 心态管理：还在练

- 连续卡题会焦虑，尤其看见别人 5 分钟 AC 的时候。
- 有时会怀疑自己是不是不适合算法。
- 但睡一觉以后还能继续写，这点我觉得还行。

## 我的刷题工作流（菜鸟保命版 2.0）

为了减少“看懂题解却不会写”的假进步，我给自己定了流程：

1. **硬想 20~40 分钟（先别跪）**
   - 先自己试，不会也要留下思路草稿。
   - 优先想暴力解，先有再优。

2. **手推样例 + 边界清单**
   - 至少自己手推 1~2 个样例。
   - 列出会翻车的边界：空输入、最小输入、重复元素、极端值。

3. **写第一版（不求优雅，先求正确）**
   - 允许代码笨一点，但逻辑要尽量自洽。
   - 报错就定位，别一上来就全盘重写。

4. **看题解（带着问题看）**
   - 重点不是“答案是什么”，而是“关键转折是什么”。
   - 记录我没想到的点，尤其是复杂度优化思路。

5. **二刷重写（隔天复现）**
   - 不看原答案重新写。
   - 能写出来才算真正会，不然只是“看过”。

6. **错因归档（痛但有效）**
   - 这题为什么卡？
   - 卡在思路、实现、边界，还是语法细节？
   - 下次如何更早规避？

## 菜鸡常见翻车清单（我本人实测）

下面这些坑我几乎都踩过：

- 把下标当值、把值当下标。
- 循环终止条件少一个等号或多一个等号。
- 只考虑“正常路径”，没考虑空输入。
- 递归没写好出口，直接把自己送进无限递归。
- 以为时间复杂度是 O(n)，实际写成了 O(n^2)。
- 提交前忘了删调试代码。

每次踩完坑我都很崩溃，
但现在我会强迫自己把坑记下来，至少别在同一个地方摔三次。
